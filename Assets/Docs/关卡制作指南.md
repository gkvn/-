# 关卡制作指南

## 一、创建新关卡场景

1. **File → New Scene**，选 Basic 2D，保存到 `Assets/Scenes/`（如 `Level_01.scene`）
2. **Build Settings** 里把新场景拖进 Scenes In Build
3. MainMenu 场景的 **LevelManager** 物体上，把场景名加到 `Level Scenes` 列表（按顺序排列）

---

## 二、场景必须包含的核心物体

### 2.1 GameCamera

| 组件 | 说明 |
|------|------|
| Camera | Projection = Orthographic |
| TopDownCamera | 挂上即可 |

**配置项（Inspector）**：

| 参数 | 位置 | 说明 |
|------|------|------|
| Orthographic Size | Camera 组件 | 可视范围，值越大看到越多。直接在 Camera 组件上调 |
| Position | Transform | 关卡画面中心位置 |
| Light Background | TopDownCamera | 亮灯阶段背景色 |
| Dark Background | TopDownCamera | 黑灯阶段背景色 |

> Scene 视图里调好的 Position 和 Orthographic Size 在 Play 时会保持。

### 2.2 LevelPhaseManager

空物体，挂 `LevelPhaseManager` 脚本。

| 参数 | 说明 |
|------|------|
| Top Down Camera | 拖入 GameCamera 物体 |

### 2.3 LevelConfig

空物体，挂 `LevelConfig` 脚本。

| 参数 | 说明 |
|------|------|
| Available Icons | 本关卡左侧画布可用的标记图标 Sprite 列表 |

### 2.4 UICanvas

从已有关卡复制即可，包含：左侧画布（LeftPanel + DrawingCanvas + IconToolbar）、右侧游戏面板（RightPanel）、通关面板（LevelCompletePanel）、阶段文字提示。

### 2.5 EventSystem

场景中需要一个 EventSystem（从模板复制或 Unity 自动创建）。

## 三、关卡场景元素

### 3.1 Player（玩家）

从 Prefab 拖入或复制已有关卡的 Player。**Tag 必须设为 `Player`。**

需要组件：Rigidbody2D、Collider2D、Animator（可选）、PlayerController。

**Inspector 可调参数**：

| 参数 | 分组 | 说明 | 默认值 |
|------|------|------|--------|
| Move Speed | Movement | 移动速度 | 5 |
| Projectile Prefab | Combat | 子弹预制体（空则运行时自动生成） | — |
| Hold Threshold | Combat | 长按多久算蓄力子弹（秒） | 0.25 |
| Bullet Spawn Offset | Combat | 子弹出生点相对角色偏移 | (0, 0.5) |
| Light Sprite | Appearance | 亮灯阶段角色贴图（不配则不切换） | — |
| Dark Sprite | Appearance | 黑灯阶段角色贴图（不配则不切换） | — |
| Charge Bar Offset | Charge Indicator | 蓄力条相对角色的位置 | (0, -0.8) |
| Charge Bar Width | Charge Indicator | 蓄力条满时的宽度 | 0.8 |
| Charge Bar Height | Charge Indicator | 蓄力条高度 | 0.1 |
| Charge Bar Sprite | Charge Indicator | 蓄力条贴图（留空则自动生成黄色纯色） | — |
| Charge Bar Charging Color | Charge Indicator | 蓄力中颜色 | 黄色 |
| Charge Bar Ready Color | Charge Indicator | 蓄力满颜色 | 浅蓝色 |
| Interact Range | Interaction | 按 E 交互的检测半径 | 1.2 |
| Prompt Offset | Interaction | 交互提示图标相对角色偏移 | (0, 1.2) |
| Death Light Radius | Death Effect | 阶段2死亡光圈初始半径 | 1.5 |
| Death Shrink Duration | Death Effect | 阶段2光圈收缩时长（秒） | 1 |

**子弹 Prefab 可调参数（Projectile 组件）**：

| 参数 | 分组 | 说明 | 默认值 |
|------|------|------|--------|
| Speed | — | 子弹飞行速度 | 10 |
| Lifetime | — | 子弹最大存活时间（秒） | 3 |
| Dot Sprite | 子弹贴图 | 点子弹贴图（可配置） | — |
| Line Sprite | 子弹贴图 | 线子弹贴图（可配置） | — |
| Hit Linger Time | 命中停留 | 击中后子弹停留时间（秒） | 0.4 |

**死亡表现**：
- **阶段1（亮灯）**：角色变红，闪烁 2 下（持续 1 秒），期间无法操作，完成后重生
- **阶段2（黑灯）**：角色变黑，以角色为中心出现圆形光圈照亮周围场景（仅光圈内可见），光圈在设定时间内收缩至消失，完成后重生

### 3.2 Wall（墙壁）

**创建方式**：
1. 创建空物体 `Wall`，下面按需创建子物体
2. 每个墙壁子物体需要：SpriteRenderer + BoxCollider2D（**isTrigger = false**）
3. 建议分组：`edge_walls`（四面边界墙）、`other_walls`（内部墙）

**黑夜隐藏**：添加 `DarkPhaseHideable` 组件，勾选 `Keep Collider When Hidden`（隐藏视觉但保留碰撞）。

### 3.3 SpawnPoint（出生点）

空物体，挂 `SpawnPoint` 脚本。Scene 视图中显示为绿色圆圈。

**每个关卡必须有且仅有一个。** 放在玩家起始位置。

### 3.4 ExitPoint（终点/出口）

需要：Collider2D（**isTrigger = true**）+ `ExitPoint` 脚本 + `DarkPhaseHideable`。

**功能**：
- 亮灯阶段：玩家走到终点 → 进入黑灯阶段
- 黑灯阶段：玩家走到终点 → 关卡通关

### 3.5 Spike（刺/陷阱）

需要：SpriteRenderer + Collider2D（**isTrigger = true**）+ `Spike` 脚本 + `DarkPhaseHideable`。

**功能**：玩家碰到立即死亡。不挡子弹（Collider 是 trigger）。

### 3.6 PressureButton（按钮）+ Door（门）

**按钮**：SpriteRenderer + Collider2D（**isTrigger = true**）+ `PressureButton` 脚本 + `DarkPhaseHideable`。

| 参数 | 说明 |
|------|------|
| Linked Door | 拖入对应的门物体 |

**门**：SpriteRenderer + BoxCollider2D + `Door` 脚本 + `DarkPhaseHideable`。
（Door 脚本会自动添加 NavMeshObstacle 并配置为 Box 形状）

**功能**：玩家靠近按钮按 E → 门打开/关闭。黑夜重置时门恢复关闭。关着的门挡玩家、挡怪物、挡子弹。

**交互范围**：由 Player 的 `Interact Range` 决定（默认 1.2）。想让按钮更容易按到就调大这个值。

### 3.7 Monster（怪物）

需要：SpriteRenderer + Collider2D（**isTrigger = false**）+ Rigidbody2D + `Monster` 脚本。
（Monster 脚本会自动添加 NavMeshAgent）

**Inspector 可调参数**：

| 参数 | 分组 | 说明 | 默认值 |
|------|------|------|--------|
| Required Combo | Combo Sequence | 击败所需子弹组合（Dot/Line 数组） | [Dot, Dot, Line] |
| Detect Range | AI | 索敌半径（Scene 视图洋红色圆圈） | 5 |
| Chase Speed | AI | 追踪速度 | 2 |
| Health Bar Offset | Health Bar | 血条/打击提示相对怪物的位置 | (0, 1.4) |
| Health Bar Width | Health Bar | 血条宽度 | 1.2 |
| Health Bar Height | Health Bar | 血条高度 | 0.15 |

**行为规则**：
- **亮灯阶段**：不移动，无敌（子弹无效），头顶显示打击提示（需要什么 combo）
- **黑灯阶段**：玩家进入索敌范围后追踪（NavMesh 寻路），碰到玩家则玩家死亡，头顶显示血条
- 打对 combo 中的一步 → 血条减少，橙红色命中特效，怪物闪红
- 打错 → combo 重置，血条恢复满，灰色特效，怪物闪红
- 完成整个 combo → 怪物消失
- 每次子弹命中怪物都会闪红

**击中特效**：
- 击中怪物（打对）：橙红色扩散
- 击中怪物（打错）：灰色扩散
- 击中墙壁：白色扩散
- 子弹命中后会停留 `Hit Linger Time` 秒后消失

### 3.8 GlowPoint（发光点）

空物体，挂 `GlowPoint` 脚本。

**特性**：黑夜中也可见（不加 DarkPhaseHideable），没有碰撞体，不挡子弹。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| Glow Color | 发光颜色 | 暖黄色 |
| Glow Radius | 光晕半径 | 1.5 |
| Core Size | 核心大小 | 0.3 |
| Pulse Amount | 呼吸幅度 | 0.1 |
| Pulse Speed | 呼吸速度 | 2 |

### 3.9 DarkPhaseHideable（黑夜隐藏组件）

添加到需要在黑夜阶段隐藏的物体上。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| Keep Collider When Hidden | 隐藏时是否保留碰撞体 | true |

- 墙壁/刺/门：勾选（隐藏视觉但保留碰撞）
- 终点/按钮：勾选

## 四、NavMesh 寻路配置

怪物使用 NavMesh 寻路。**每次修改墙壁或门的布局后都需要重新烘焙。**

1. 点 **Tools → 创建并烘焙 NavMesh 2D**
   - 自动创建 NavFloor（地面碰撞体）
   - 自动标记 Wall 下的墙壁为不可行走
   - 自动烘焙导航网格
2. **Ctrl+S** 保存场景

**调整 Agent 半径**（影响窄通道通过性）：
- 点 **Tools → 设置 NavMesh Agent 半径**
- 拉滑条调整（越小窄通道覆盖越多，越大离墙越远）
- 点"保存并重新烘焙"

**验证**：Scene 视图中蓝色区域 = 可行走区域，确认所有通道都有覆盖。

## 五、操作说明

| 按键 | 功能 |
|------|------|
| WASD | 移动 |
| 鼠标左键点击 | 发射"点"子弹 |
| 鼠标左键长按 | 蓄力后发射"线"子弹（蓄满时角色旁有特效提示） |
| E | 与附近可交互物体交互（头顶出现提示时可按） |

## 六、游戏流程

1. **亮灯阶段（Phase 1）**：
   - 玩家从 SpawnPoint 出发，探索关卡
   - 在左侧画布拖拽图标做标记
   - 怪物不动，无敌，显示打击提示
   - 走到 ExitPoint → 进入黑灯阶段

2. **黑灯阶段（Phase 2）**：
   - 场景变暗，地形/陷阱/出口/按钮/门隐藏（碰撞仍在）
   - 玩家从 SpawnPoint 重新出发
   - 参考左侧画布标记通关
   - 怪物开始追踪，可用 combo 击败
   - GlowPoint 仍然可见
   - 走到 ExitPoint → 关卡通关

3. **死亡**：
   - 碰到 Spike 或被 Monster 碰到 → 死亡
   - 阶段1：角色变红闪烁（1秒），重生
   - 阶段2：角色变黑 + 圆形光圈照亮周围并收缩（可配置时长），重生
   - 死亡后所有交互物体重置（按钮、门、怪物回到初始状态）

## 七、作弊键（Debug）

| 按键 | 功能 | 条件 |
|------|------|------|
| F1 | 切换作弊灯光（看到场景但不改变阶段） | 仅黑灯阶段 |
| F2 | 重启当前阶段（重置物体 + 回到出生点） | 任何时候 |
| F3 | 重启整个关卡（重新加载场景） | 任何时候 |
| F4 | 跳到下一阶段（亮灯→黑灯，黑灯→通关） | 任何时候 |
| Ctrl+H | Scene 视图中隐藏/显示 UI（方便编辑） | 仅编辑器 |
